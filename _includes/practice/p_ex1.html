Solution: Complete API Documentation

Improved API Documentation:

### GET /v1/users/{userId}
Retrieves a single user's details.

**Path Parameters**:
- `userId` (integer, required): Unique identifier of the user.

**Headers**:
- `Authorization` (string, required): Bearer token for authentication.
- `Accept` (string, optional): Specifies the response format (e.g., application/json).

**Example Request**:
GET /v1/users/12345 HTTP/1.1
Host: api.example.com
Authorization: Bearer <token>
Accept: application/json

**Response: 200 OK**
{
  "id": 12345,
  "name": "John Doe",
  "email": "john@example.com"
}

**Error Responses:**
- **404 Not Found**: User ID does not exist.
  {
    "error": "User not found",
    "code": 404
  }
- **401 Unauthorized**: Invalid or missing authorization token.
  {
    "error": "Unauthorized",
    "code": 401
  }
- **429 Too Many Requests**: Rate limit exceeded.
  {
    "error": "Rate limit exceeded",
    "code": 429
  }

Key Improvements:

  • Added `Authorization` and `Accept` headers.
  • Specified `userId` as an integer and required parameter.
  • Included error handling for `404`, `401`, and `429` responses.
  • Improved response format and examples for clarity.
